# EventBridge Alerting Plan

## Status

Planned.

This document will define the planned EventBridge alerting workflow for AWS security events.

---

## Purpose

Amazon EventBridge can route AWS events to targets such as notification services, Lambda functions, or other workflows.

For this lab, EventBridge will be reviewed as a basic event-routing and alerting tool.

---

## Planned Alerting Concepts

EventBridge may be used to understand how security events can trigger response workflows.

Potential event sources:

- GuardDuty findings
- CloudTrail events
- IAM changes
- EC2 state changes
- Security group changes
- S3 configuration changes

---

## Example Alerting Use Cases

| Use Case | Event Source | Possible Action |
|---|---|---|
| GuardDuty finding created | GuardDuty | Send notification or document alert workflow |
| Security group modified | CloudTrail | Review network exposure |
| IAM policy attached | CloudTrail | Review privilege change |
| Root account activity | CloudTrail | Escalate investigation |
| EC2 instance launched | CloudTrail / EC2 event | Review resource creation |

## Detection Workflow

Planned workflow:

1. AWS activity occurs.
2. Event is recorded by CloudTrail or generated by GuardDuty.
3. EventBridge matches an event pattern.
4. EventBridge routes the event to a target.
5. Alert or response action is reviewed.
6. Investigation notes are documented.

---

## Scope Control

This repo will keep EventBridge simple.

Advanced automation belongs in later AWS automation work.

This project should focus on understanding:

- What event occurred
- Why it matters
- How it could be routed
- What investigation should follow
